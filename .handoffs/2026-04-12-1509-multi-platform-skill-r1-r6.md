# Handoff: Multi-platform skill adaptation — R1, R2, R6 landed

**Date:** 2026-04-12
**Branch:** main
**State:** Green

## Where things stand

Four PRs shipped this session against a ten-item requirements list for making `cased` work correctly across Claude Code, Codex, and other harnesses. Session began from a real failure: a Codex-driven rebar audit produced four shallow findings because the skill's "dispatch applicable agents in parallel" prose was too abstract for Codex to turn into `spawn_agent` calls. The requirements list (gitignored at `ref/multi-platform-skills-requirements.md`) was distilled from a close read of `~/source/reference/superpowers/`, which already runs cleanly under Codex.

## Decisions made

- **Living requirements doc in gitignored `ref/`.** Eventual target: compile to a Copier template that scaffolds multi-platform-ready skills. Ten requirements (R1–R10) with per-item status table.
- **One PR per requirement.** Small, reviewable, revertable.
- **R1 (PR #23)** — `skills/cased/references/codex-tools.md`: `Task` → `spawn_agent` / `wait` / `close_agent` mapping, required `multi_agent = true` in `~/.codex/config.toml`, named-agent workaround (read rubric file, fill message template with task-delegation framing + XML-tagged instructions + explicit execution directive, spawn as `worker`), troubleshooting for the "Phase 2 dispatched one agent and stopped" failure.
- **R2 (PR #24)** — Phase 2 rewritten in direct-imperative language: every dispatch is a named, file-linked imperative (`Dispatch the \`security\` subagent (agents/security.md).`). "Issue every dispatch before the first wait" stated verbatim in both domain-skill and no-domain-skill blocks. Corrected crustoleum agent names to match actual files (`safety-auditor`, `api-type-design`, `error-robustness`, `concurrency-reviewer`, `supply-chain-deps`, `performance-reviewer`, `completeness`).
- **R6 (PR #25)** — Status protocol: factored the ~35-line duplicated Output Format block out of all 8 agent rubrics into `skills/cased/references/subagent-output-contract.md`. Canonical envelope is `status: DONE | DONE_WITH_PARTIAL_COVERAGE | BLOCKED | NEEDS_CONTEXT` with exactly one of `findings` or `blocker`. Per-status controller behaviour documented. Each rubric kept only its surface-specific signals (criterion prefix, surface name, `evidence_lang` default, domain-specific "what counts as a finding" admonition). Net: −222 lines of duplication, one canonical reference. Also fixed a real bug in R1: codex-tools.md's dispatch template was specifying a `narrative:` envelope the rubrics never produce (cased SKILL.md explicitly says thesis/verdict are controller-written). Envelope now matches rubric output.
- **R4 and R9 closed as covered by R1.** Named-agent workaround and explicit config requirements both live in `codex-tools.md`.

## Landmines

- **`ref/` is gitignored globally** via `~/.gitignore` line 38, not this repo's `.gitignore`. Any `ls -la` won't show it as ignored-here. Don't try to stage files from `ref/` — they exist for iteration, never commit.
- **File inventory sources: not all files live in `src/`.** `skills/cased/agents/*.md`, `skills/cased/references/codex-tools.md`, `skills/cased/references/subagent-output-contract.md`, and `skills/cased/SKILL.md` are authored directly in `skills/cased/` with no `src/` counterpart. Only `findings.schema.json` / `recon.schema.json` / `findings-schema.yaml.md` / `recon-schema.yaml.md` / `findings.example.yaml` / `recon.example.yaml` get regenerated by `scripts/build-viewer.sh`. Editing a rubric does not require a rebuild; editing anything under `src/schemas/` does. `just check-bundle` enforces the invariant.
- **Agent frontmatter `name:` can differ from filename.** `skills/cased/agents/reviewer.md` declares `name: audit-reviewer`. When dispatching, always use the frontmatter name (for Claude Code) and the file path (for Codex) together: `Dispatch the \`audit-reviewer\` subagent (agents/reviewer.md)`. Same rule applies to crustoleum: `concurrency.md` is `concurrency-reviewer`, `performance.md` is `performance-reviewer`, `safety-auditor.md` is `safety-auditor`.
- **Codex subagents require opt-in.** Users must add `[features]\nmulti_agent = true` to `~/.codex/config.toml` before the skill's Phase 2 dispatches work. Without it, `spawn_agent` isn't available and the session silently degrades to single-context analysis. Troubleshooting entry in `codex-tools.md` catches this if it happens.
- **Superpowers does not accept "skills compliance" PRs.** Their CLAUDE.md says restructuring/reformatting skills to match Anthropic's published guidance will not be accepted without extensive eval evidence. If any cased change patterns reverse-apply to a superpowers PR, do not open it.
- **The superpowers "94% PR rejection rate" is maintainer self-report**, not a measured GitHub stat. Cite with that qualifier.
